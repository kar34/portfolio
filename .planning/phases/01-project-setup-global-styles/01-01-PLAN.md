  <action>
1. Update `portfolio/astro.config.mjs` to use the Tailwind v4 Vite plugin. This is the ONLY correct approach for Tailwind v4 with Astro — do NOT use @astrojs/tailwind (which is v3 only) and do NOT create a tailwind.config.js (v4 uses CSS-based configuration):

```js
import { defineConfig } from 'astro/config';
import tailwindcss from '@tailwindcss/vite';

export default defineConfig({
  vite: {
    plugins: [tailwindcss()],
  },
});
```

2. Create `portfolio/src/styles/global.css` with Tailwind v4 syntax. In v4, the entry point is `@import "tailwindcss"` (replaces the old `@tailwind base/components/utilities` directives). Theme customization uses `@theme` blocks in CSS (no separate tailwind.config.js needed):

```css
@import "tailwindcss";
@import url('https://fonts.googleapis.com/css2?family=Poppins:wght@400;500;600&display=swap');

@theme {
  --font-poppins: 'Poppins', sans-serif;
}

/*
 * Global base styles for Kenny Raymond Portfolio
 * Design system: pure black bg, pure white text, Poppins font, no visible scrollbar
 */
@layer base {
  html {
    font-family: 'Poppins', sans-serif;
    background-color: #000000;
    color: #ffffff;
    scrollbar-width: none; /* Firefox */
  }

  html::-webkit-scrollbar {
    display: none; /* Chrome, Safari, Edge */
  }

  body {
    margin: 0;
    padding: 0;
    min-height: 100vh;
    -webkit-font-smoothing: antialiased;
    -moz-osx-font-smoothing: grayscale;
  }

  a {
    color: inherit;
    text-decoration: none;
  }

  img {
    max-width: 100%;
    height: auto;
  }
}
```

Key points about v4 configuration:
- `@import "tailwindcss"` is the single entry point (replaces `@tailwind base; @tailwind components; @tailwind utilities;`)
- `@theme {}` block registers custom theme values as CSS custom properties (replaces tailwind.config.js `theme.extend`)
- No tailwind.config.js file is needed
- The Google Fonts @import must come AFTER the Tailwind import
